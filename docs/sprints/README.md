**<h2>Estrutura dos Relatórios</h2>**

Para garantir a rastreabilidade, a transparência e a padronização do nosso trabalho, todos os relatórios de Sprint seguem a seguinte documentação:

1. **Objetivos da Sprint:** Metas traçadas para o ciclo.
2. **Entregas Coletivas:** Funcionalidades, issues fechadas e melhorias entregues em conjunto.
3. **Contribuições Individuais:** O que cada membro desenvolveu (com links diretos para os Diários de Bordo, PRs e Issues).
4. **Maiores Avanços:** Destaques, conquistas da equipe e impactos na comunidade.
5. **Maiores Dificuldades:** Barreiras técnicas, de coordenação ou de comunicação enfrentadas.
6. **Lições Aprendidas:** Conhecimentos adquiridos que otimizarão os próximos passos.
7. **Planejamento Futuro:** Definição do backlog e foco para a próxima iteração.
